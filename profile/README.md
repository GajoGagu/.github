### 📝 Git Commit Message 규칙

모든 커밋 메시지는 다음 규칙을 따라 작성해주세요:

```

<type>: <간결한 설명>

[선택] 본문 (한 줄 띄우고 자세한 설명)

````

#### 📌 type 목록

| type      | 설명                          |
|-----------|-------------------------------|
| feat      | 새로운 기능 추가               |
| fix       | 버그 수정                     |
| refactor  | 코드 리팩토링 (기능 변화 없음) |
| style     | 코드 포맷팅, 세미콜론 등       |
| chore     | 빌드/배포 설정 수정 등 기타    |
| docs      | 문서 작성 또는 수정            |
| test      | 테스트 코드 추가 또는 수정     |

#### ✅ 예시

```bash
feat: 로그인 화면 UI 추가

refactor: API 응답 구조 정리

fix: 이미지 업로드 오류 해결
````
